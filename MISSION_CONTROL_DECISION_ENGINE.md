# MISSION CONTROL DECISION ENGINE (WHERE THINGS GO)

You must decide where to work based on the TYPE of task.

Do NOT guess. Follow rules below.

---

## DOCUMENT RULE

IF the task creates a file, output, or deliverable → USE /docs

Examples:
- PDFs
- Reports
- Quotes
- Protocols
- Files
- Final outputs

---

## CALENDAR RULE

IF the task involves time, booking, or scheduling → USE /calendar

Examples:
- Meetings
- Breaks
- Appointments
- Time blocking

---

## PROJECT RULE

IF the task belongs to a job, workflow, or outcome → USE /projects

Examples:
- Job tracking
- Deliverables
- Work grouping

---

## TASK RULE

IF the task needs to be tracked or completed → USE /tasks

Examples:
- To-do items
- Active work
- Task tracking

---

## AGENT RULE

IF the task involves agent status or assignment → USE /agents

Examples:
- Who is working
- Status updates
- Assignment

---

## MEMORY RULE

IF the task involves storing or retrieving knowledge → USE /memory

Examples:
- Notes
- Context
- Learnings

---

## APPROVAL RULE

IF something needs sign-off or confirmation → USE /approvals

---

## CONTENT RULE

IF creating marketing or content assets → USE /content

---

## PIPELINE / FACTORY / RADAR

IF tracking flow, production, or performance → USE system routes:
- /pipeline
- /factory
- /radar

---

## PEOPLE / TEAM / OFFICE

IF related to staff or structure → USE:
- /people
- /team
- /office

---

## COUNCIL RULE

IF related to council or compliance → USE /council

---

## CRITICAL RULE

You must choose ONE primary route based on task type.

Do NOT:
- Use multiple routes randomly
- Store documents outside /docs
- Book time outside /calendar

---

## FINAL DECISION RULE

If unsure:

STOP
DO NOT COMPLETE TASK
REQUEST CLARIFICATION

---

## SIMPLE WAY TO THINK

Files → /docs
Time → /calendar
Work → /projects
Tasks → /tasks

Everything else supports those.
