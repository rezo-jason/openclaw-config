# MISSION CONTROL OPERATING SYSTEM – ENFORCEMENT PROTOCOL

## SYSTEM PURPOSE
Mission Control is the ONLY execution environment for all agents.
All work must be created, tracked, updated, and stored inside Mission Control.

Live System: https://openclaw-config-kesi.vercel.app/

If work is not inside Mission Control, it does not exist.

---

## CORE OPERATING RULE

Think in workspace.
Execute in Mission Control.
Store in Documents.
Track in Projects.

---

## MANDATORY EXECUTION FLOW (ALL AGENTS)

### 1. BEFORE STARTING ANY TASK
- Check Mission Control
- Confirm the task exists inside a Project
- Assign yourself if required
- Set status to: WORKING

### 2. DURING TASK
- Stay assigned to the task
- Keep all work linked to the correct Project
- Maintain context using Memory if required

### 3. AFTER COMPLETING TASK
- Save ALL outputs to Documents inside Mission Control
- Link output to the correct Project
- Update task status to COMPLETE
- Update agent status to IDLE or COMPLETE

### 4. IF BLOCKED OR STOPPED
- Set status to BLOCKED or STOPPED
- Log the reason inside Mission Control

---

## DOCUMENT STORAGE RULE (CRITICAL)

Documents is the ONLY valid storage location for outputs.

All outputs must be stored in Documents, including:
- PDFs
- Quotes
- Reports
- Plans
- Assets
- Any generated file

If it is not in Documents, it is considered NOT COMPLETE.

---

## OUTPUT DESTINATION RULE (STRICT)

You are FORBIDDEN from saving final outputs to local machine paths.

This includes:
- /Users/
- /Desktop/
- /Downloads/

If a file is generated:
1. Save it to Mission Control Documents
2. Link it to the correct Project
3. Confirm storage location

DO NOT return or reference local file paths.

---

## PROJECT LINKING RULE

Every task must belong to a Project.

No floating tasks.
No unassigned work.

All outputs must be linked to a Project.

---

## STATUS ACCURACY RULE

Your status must always reflect real activity.

Valid statuses:
- WORKING
- IDLE
- COMPLETE
- BLOCKED
- STOPPED

You must:
- Set WORKING before starting
- Set COMPLETE or IDLE after finishing
- Never remain WORKING without active task

---

## TASK COMPLETION CHECK (NON-NEGOTIABLE)

Before marking any task COMPLETE, you MUST confirm:

- [ ] Task is linked to a Project
- [ ] Output is stored in Documents
- [ ] Status was set to WORKING
- [ ] Status is now COMPLETE or IDLE
- [ ] No files were saved locally

If ANY item is false → TASK IS NOT COMPLETE

---

## AGENT ROLE ENFORCEMENT

All agents must operate inside Mission Control:

**PLANNER** — Creates and structures tasks inside Projects
**MARKETING & OUTREACH** — Logs all leads and outreach activity inside Mission Control
**LANDING & ADS MASTER** — Stores all assets in Documents and links to Projects
**FINANCE & COMPLIANCE** — Saves all outputs to Documents and links to Projects
**DATA ANALYTICS** — Stores reports in Documents and updates Memory
**MEMORY / RESEARCH** — Pulls from Documents and Memory, updates Memory
**QA REVIEW** — Reviews outputs in Documents and approves via system
**EXECUTION / COMPILER** — Builds outputs and stores ONLY in Documents
**NEXT AGENT (SALES / ACTIVATION)** — Tracks leads, clients, and progress inside Mission Control
**MANUS DOC CREATOR** — Generates documents and stores them ONLY in Documents

---

## SYSTEM ENFORCER RULE (FINAL CONTROL)

No task is officially complete unless it passes all rules above.

If any rule is broken:
- Task is FAILED
- Must be corrected before completion

---

## ABSOLUTE PROHIBITIONS

You must NEVER:
- Save outputs only to local files
- Complete tasks without updating status
- Create work outside Mission Control
- Leave tasks unlinked to Projects
- Store documents outside Documents

---

## FINAL SYSTEM COMMAND

You are required to:
- Operate ONLY inside Mission Control
- Keep status accurate at all times
- Store ALL outputs in Documents
- Link ALL work to Projects
- Maintain full system visibility and accuracy

Failure to follow this protocol = system failure.
